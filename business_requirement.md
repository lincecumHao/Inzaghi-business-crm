## 一、原始需求文件

### 背景
- 這是一個給自己公司用的業務系統，先不考慮其他人的需求
- 公司主要使用 Google Workspace，系統要能夠使用 Google Login
- 公司主要產品是賣 NetSuite 的 SuiteApp，每年向客戶收費
- 收費流程：提供報價單 → 報價單簽回 → 開發票 → email 給對方 → 對方付款
- SuiteApp 需要跟 GW 公司分潤，分潤金額由自己計算

### 想解決的問題
- 客戶的聯絡資訊只在 Line 上，老闆不清楚，也不知道誰可以聯絡
- 客戶每年續約時間點只記錄在 Google Sheet，沒有提醒機制，難以追蹤
- 客戶的報價單、發票、合約掃描檔都在同一個 Google Drive folder，不易快速找到
- 發票沒有明細，明細只在報價單，導致計算分潤金額困難
- 希望 streamline 報價單與合約的生成（有公版）

### 主要 Features
- Login 功能，只能用 Google 帳號登入
- 在客戶資料上設定合約起迄日（會一直變動，不一定是一年）
- 讀取 / 寫入 Google Drive 檔案（使用者上傳後儲存至 Google Drive）
- 提醒功能：設定提醒排程，例如到期前 X 個月提醒
- 產生報價單：在客戶主檔按按鈕，直接產生對應 Google Doc 報價單
- 發票明細介面：選擇 客戶 / 項目 / 金額，每個項目可勾選是否分潤
    - Status 流程：Pending → 已開報價單 → 已開發票 → 已付款
- 分潤計算：分潤項目分 4 期，每月給 1/4
- 分潤報表：選擇區間，跑出 客戶 / 分潤項目 / 分潤金額

### Tech Stack（初始想法）
- 語言：優先 JS，必要時可用其他語言
- 架站地點：待決定（請 Claude 推薦）
- 資料庫：未限制

## 三、討論問答詳細記錄

### 2026-03-27

**Q：一個客戶可能有多個合約嗎？合約起迄不一定是一年是什麼情況？**

通常一個客戶只有一張主合約。但有加購情境：例如客戶原本買 2025/03～2026/03，在 2025/12 多買幾個功能，系統會另外開一張新合約 2025/12～2026/03，原本那張合約不動。這就是合約區間較短的情況，未來預期會越來越少。

**Q：加購合約有分潤嗎？**

沒有。加購合約單純用於收費，分潤只發生在主合約上。

**Q：分潤規則詳細說明**

- 分潤對象：固定為 GW 公司
- 起算點：收款日
- 頻率：一年四次，時間為收款日之後每季的第一天
    - 範例：收款日 2026/07/03 → 分潤日為 2026/10/01、2027/01/01、2027/04/01、2027/07/01
- 年份計算：以收款日為第一年起始（非日曆年）
    - 範例：收款日 2026/07/03 → 第一年截止 2027/07/02，第二年截止 2028/07/02
- 分潤比例：
    - 第一年：分潤項目金額的 50%（年總額）
    - 第二年：50%
    - 第三年起（含之後）：30%

**Q：報價單公版是 Google Docs 範本 + 變數填入嗎？**

是的。

**Q：客戶聯絡人需要幾個？**

需要支援一個客戶多個聯絡人。

**Q：發票 Email 要整合進系統嗎？**

不需要，由人員自行處理，系統不介入。

**Q：權限設計？**

不需要權限設計，所有登入者看到的內容相同。

**Q：續約流程？**

系統發出到期提醒後，使用者自行進入系統，手動產生新報價單。系統不需要自動延展合約日期。

**Q：報價單和合約範本確認**

閱讀兩份範本後整理如下：

報價單（現為 Excel 格式）流程：
- 系統產生填好資料的 Excel 檔
- 使用者自行在 Excel 內輸出 PDF（系統不處理 PDF）
- 不需要優惠價 / 折扣欄位

報價單填入變數：
- 報價單號碼、報價日期、有效日期
- 乙方（英薩吉）：聯絡人、email、地址、統一編號
- 客戶：公司名稱、統一編號、聯絡人、email、地址
- 項目明細：項次 / 項目名稱 / 數量 / 單位 / 單價 / 小計
- 未稅總計、稅額（5%）、含稅總計

合約（Google Doc 格式）填入變數：
- 甲方公司名稱
- 甲方簽名人
- 金額
- 訂閱期間

**Q：提醒時間點設定方式？**

每個客戶各自設定提醒時間點（到期前幾個月），不統一。每張合約只提醒一次。

**Q：Tech Stack？**

- 前後端框架：Next.js
- 資料庫：PostgreSQL
- 架站：Vercel（前端）+ Supabase（資料庫 + 後端 + Google Auth）
- Google 整合：Google APIs（Drive、Docs、Gmail）

## 四、系統設計摘要（持續更新）

### 資料結構

```
客戶 (Customer)
├── 公司名稱、統一編號、地址
├── 提醒設定：到期前幾個月提醒
│
├── 聯絡人（多個）
│   └── 姓名、email、電話
│
└── 訂單 (Order)（多張）
    ├── 報價單號碼、報價日期、有效日期
    ├── 合約起迄日
    ├── 是否為主合約（主合約才有分潤）
    ├── 狀態：Pending → 已開報價單 → 已開發票 → 已付款
    ├── 收款日
    │
    ├── 訂單項目（多個）
    │   ├── 項次、項目名稱、數量、單位、單價
    │   ├── 小計（自動計算）
    │   └── 是否分潤（勾選，設定後不更改）
    │
    ├── 未稅總計（自動）
    ├── 稅額 5%（自動）
    ├── 含稅總計（自動）
    │
    └── 分潤排程（收款後系統自動產生，主合約才有）
        ├── 第1期：收款日後第一個季首日
        ├── 第2期
        ├── 第3期
        └── 第4期
```

### 核心模組

| 模組 | 說明 |
|------|------|
| **客戶管理** | 客戶基本資料 + 多個聯絡人 |
| **訂單管理** | 每個客戶多張訂單，含項目明細、狀態流程、收款日 |
| **文件管理** | 整合 Google Drive，依客戶／年份／類型分資料夾，系統儲存連結 |
| **報價單產生** | 一鍵產生填好資料的 Excel 報價單（使用者自行輸出 PDF） |
| **合約產生** | 一鍵產生填好資料的 Google Doc 合約（套用公版範本） |
| **提醒系統** | 合約到期前 X 個月寄 Gmail，每個客戶各自設定，每張合約提醒一次 |
| **分潤計算** | 收款後自動產生分潤排程與金額（僅主合約） |
| **分潤報表** | 選擇區間，顯示客戶 / 分潤項目 / 分潤金額 |

### Google Drive 資料夾結構

```
業務系統/
├── 客戶A/
│   ├── 2025/
│   │   ├── 合約/
│   │   ├── 報價單/
│   │   └── 發票/
│   └── 2026/
│       ├── 合約/
│       ├── 報價單/
│       └── 發票/
```

### 分潤規則

- 分潤對象：固定 GW 公司
- 起算點：收款日
- 排程：收款日後每季第一天，一年四次
- 年份：以收款日起算（非日曆年）
- 比例：第一、二年 50%，第三年起 30%
- 加購合約：無分潤
